## TL;DR
Shafik Quoraishee and Joanne Song from The New York Times present experimental work on running agentic AI locally on mobile devices for games—covering game-playing agents, constraint optimization, and dynamic accessibility. The vision is a future where small, on-device AI models act as empathetic, real-time partners that adapt gameplay and accessibility to each individual player without needing cloud round trips.

## Key Takeaways
- NYT puzzles are made by people, not AI; this talk covers experimental, local agentic work around solvability and playability—not game generation.
- Local (on-device) AI offers lower latency, privacy, offline capability, and personalization compared to cloud-based AI.
- Traditional game AI used finite state machines (e.g., Pac-Man) and reinforcement learning (e.g., AlphaGo, EfficientZero); agentic systems instead use in-context reasoning and tool calls without changing model weights or requiring reward loops.
- On-device agents face three major constraints: space (model weights, state history, render headroom), time (must fit within ~16 ms per frame at 60 Hz), and energy (battery life).
- A constraint graph manages these competing demands with soft and hard penalties to prioritize user experience (e.g., time overruns are penalized more than minor space overages).
- Gaze estimation models and Arc AGI benchmarks are relevant to advancing agent intelligence and input understanding on devices.
- A constraint satisfaction agent can solve NYT Mini Crosswords using natural language and backtracking over a constraint graph.
- Accessibility is grounded in WCAG 2.2 (perceivable, operable, understandable, robust), with WCAG 3.0 moving toward graded (bronze/silver/gold) scoring instead of binary pass/fail.
- An on-device accessibility agent can dynamically adjust input tolerance, step granularity, and layout sizing in real time based on observed player behavior (gaze, shaky taps, focus traps).
- Future needs include sub-16 ms planning, predictive layout modeling, long-term personal memory, a shared game-state language across games, and better chips with honest benchmarks.

## Detailed Breakdown

### Disclaimers and Context [00:01](https://www.youtube.com/watch?v=418t26CVz-w&t=1s)
Shafik Quoraishee opens with disclaimers: NYT puzzles are made by people, not AI, and there are no AI features in the games themselves (Wordle Bot is not an AI feature). The talk covers experimental work exploring agent solvability and playability capabilities on local devices—not game generation.

### History of AI in Games [02:03](https://www.youtube.com/watch?v=418t26CVz-w&t=123s)
AI has been used in games since the 1980s, starting with finite state machines in games like Pac-Man. This symbolic, conditional AI evolved into reinforcement learning approaches, culminating in modern systems like AlphaGo, AlphaZero, and the sample-efficient EfficientZero models.

### The Case for Local On-Device AI [03:05](https://www.youtube.com/watch?v=418t26CVz-w&t=185s)
Most AI infrastructure today runs in the cloud, which introduces latency and cost. Local on-device computation offers faster response times (within frame computation), privacy (data never leaves the device), offline capability (works in subway tunnels), and personalization tailored to the local user.

### Agentic AI vs. Reinforcement Learning [05:08](https://www.youtube.com/watch?v=418t26CVz-w&t=308s)
Reinforcement learning trains a model by changing its weights over many iterations to tune it to a specific game. Agentic systems, by contrast, use language models with in-context learning to reason over game state in real time—no reward system, no weight changes. The agent uses an agentic loop (perceive, predict, decide, act) and adapts dynamically using tool calls and local device functionality.

### Agentic Space Invaders Demo [07:15](https://www.youtube.com/watch?v=418t26CVz-w&t=435s)
An agentic version of Space Invaders was built using a local agentic framework. The agent perceives the scene, predicts enemy moves, decides actions (e.g., shooting or dodging), and loops. The agent's internal states are visible on the right side of the demo, showing the perceive-predict-decide-act cycle running on an on-device model.

### Constraints: Space, Time, and Energy [08:17](https://www.youtube.com/watch?v=418t26CVz-w&t=497s)
On-device agents face three constraint categories. Space: fitting model weights, compressed state history, planning artifacts, and render headroom. Time: at 60 Hz, each frame is 16 ms; agent planning must fit within this budget or the game exhibits jank. Energy: current devices aren't optimized for agentic workloads, though NPUs and AI chips are improving. A constraint graph balances these with soft and hard penalties—time overruns are penalized more heavily to protect user experience.

### Gaze Estimation and Arc AGI [10:21](https://www.youtube.com/watch?v=418t26CVz-w&t=621s)
Gaze estimation models (CNNs that translate gaze data into agentic behavior) are becoming viable inputs. The Arc AGI benchmark is relevant because base model intelligence must improve for agents to handle complex game decisions in the future.

### Constraint Satisfaction Agent for Mini Crosswords [11:23](https://www.youtube.com/watch?v=418t26CVz-w&t=683s)
A constraint satisfaction agent was built to solve NYT Mini Crosswords. It uses a satisfaction graph to place words, backtracking efficiently when constraints become too crowded or incorrect. The agent can use natural language from the graph to find the correct crossword layout.

### Accessibility: WCAG Standards and Pitfalls [12:25](https://www.youtube.com/watch?v=418t26CVz-w&t=745s)
Joanne Song shifts to accessibility. Traditional games use fixed state models with static menus that are deterministic and blind to real-time needs. The work is grounded in WCAG 2.2's four pillars: perceivable, operable, understandable, robust. Common pitfalls in mobile crosswords include tiny grid cells causing mistaps, invisible highlights for low-vision players, and unforgiving timers. WCAG 3.0 (in draft) moves toward graded bronze/silver/gold scoring instead of binary pass/fail.

### Dynamic Accessibility Dials [14:30](https://www.youtube.com/watch?v=418t26CVz-w&t=870s)
Core accessibility dials scale with player needs in real time: input tolerance (forgiveness for physical constraints), step granularity (breaking complex tasks into smaller steps), and timing. The agent's job is to dynamically calculate these dials as they rise and fall together, balancing against the player's live context using on-device vision and interaction models.

### Real-Time Accessibility Agent Actions [15:01](https://www.youtube.com/watch?v=418t26CVz-w&t=901s)
The agent doesn't just observe—it acts. Examples include: tracking eye gaze to detect search friction, analyzing shaky taps for motor difficulties, enabling handwriting recognition, monitoring focus paths to detect and fix keyboard/focus traps by injecting exit routes, and dynamically resizing too-small touch targets by rewriting the layout live. The system becomes a real-time tug-of-war, tuning the game to the player rather than the reverse.

### Future Roadmap [16:34](https://www.youtube.com/watch?v=418t26CVz-w&t=994s)
For local agents to truly understand games, several advances are needed: faster planning within 16 ms frames, predictive models to preview layout changes before applying them, long-term memory to learn individual habits, a shared game-state language so one agent works across multiple games, and better chips paired with honest benchmarks. The future of AI doesn't have to be one centralized brain—it can be billions of small local brains, each shaped by the individual it serves.
