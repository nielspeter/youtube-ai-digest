## TL;DR
Steve Yegge delivers an urgent, deliberately fear-inducing talk on agentic security, arguing that AI-accelerated coding is dramatically expanding the vulnerability surface through both traditional defects and novel attack vectors like "slop squatting." He advocates for integrating security tools (like Snyk and Chainguard) directly into LLM coding workflows as dedicated multi-pass review steps, while warning that the broader threat landscape—including AI-driven social engineering—is arriving faster than most people realize.

## Key Takeaways
- **The defect surface is expanding exponentially:** If AI enables 10x faster coding and the security defect rate stays the same (or worsens), the absolute number of vulnerabilities skyrockets.
- **New attack classes are emerging:** "Slop squatting"—where attackers register hallucinated package names that LLMs invent—creates backdoors that are extremely difficult to detect.
- **Security has no urgency half-life:** Unlike functional bugs, security vulnerabilities compound in danger over time, making early detection at the developer's fingertips critical.
- **LLMs need dedicated security passes:** You cannot ask an LLM to handle correctness and security simultaneously; it will do a poor job of both. Security must be its own explicit review pass.
- **The "rule of five":** LLMs often require up to four or five iterative review passes before their output is production-ready, similar to painting a wall with multiple coats.
- **Tool integration is key:** Because LLMs are lazy and will use tools to save tokens, you should give them security tools like Snyk (for code and dependencies) and Chainguard (for pre-vetted container images).
- **Open-source models are catching up to offensive capabilities:** Five Eyes warns that open-source models matching state-level hacking tools (referred to as "Mythos") is months away, not years.
- **Personal AI scams are imminent:** Convincing AI-generated video calls impersonating distressed family members will drain bank accounts; families should establish offline code words now.
- **Adversarial agent oversight is needed:** As organizations deploy 24/7 autonomous agents, they need adversarial supervisor agents to monitor permissions and prevent over-privileged service accounts.
- **Agentic security is a new discipline:** Companies should begin establishing dedicated roles and practices for securing always-on agents against prompt injection and other emerging threats.

### The 10x Defect Surface [01:35](https://www.youtube.com/watch?v=yWS0udrIOc8&t=95s)
Recounting a visit to a major bank, Yegge recalls their chief security architect asking a devastating question: if everyone ships code 10x faster and the defect rate stays the same, doesn't the security defect surface grow 10x? Yegge admits the reality is worse—the defect rate itself will get worse because AI is writing the code.

### Slop Squatting and New Attack Vectors [03:13](https://www.youtube.com/watch?v=yWS0udrIOc8&t=193s)
Beyond traditional vulnerabilities like cross-site scripting (which Fable actually introduced into Yegge's code), Yegge highlights "slop squatting." This occurs when an LLM hallucinates a package name, and an attacker registers that name with a backdoor. The code builds and tests pass, but a vulnerability is introduced. He asks the audience how you even detect this happening.

### Bug Urgency and the Google TAP Team [04:49](https://www.youtube.com/watch?v=yWS0udrIOc8&t=289s)
Yegge draws on his time at Google working near the Test Automation Platform (TAP) team. They learned that bugs have a "half-life of urgency"—you fix them immediately if warned while typing, but procrastinate at code review. Google moved bug reporting to the developer's fingertips. However, Yegge notes security bugs are different: their danger compounds over time, so they require this same immediate-surfacing approach.

### The Rule of Five and Dedicated Security Passes [06:56](https://www.youtube.com/watch?v=yWS0udrIOc8&t=416s)
Yegge shares that he ran a security hardening pass on his 30-year personal game project using Fable, which reported things looked good. Running Snyk afterward found 241 vulnerabilities Fable missed. He explains the "rule of five"—LLMs need up to five review passes to finalize work. Because LLMs are good at one task at a time, you cannot combine security and correctness checks; security must be a separate pass.

### Software 2.0 and Integrating Security Tools [09:31](https://www.youtube.com/watch?v=yWS0udrIOc8&t=571s)
Referencing his essay "Software 2.0," Yegge explains LLMs are lazy and will use tools to save tokens. Therefore, you should integrate tools like Snyk (for code, dependencies, and slop squatting detection) and Chainguard (for pre-vetted container images) into the LLM's workflow. He suggests having LLMs run multiple security tools and check each other's work.

### Five Eyes Warning and Open Source Models [12:08](https://www.youtube.com/watch?v=yWS0udrIOc8&t=728s)
Yegge warns that Five Eyes (an intelligence alliance) announced it is months, not years, until open-source models catch up to "Mythos" (state-level hacking capabilities). He notes the audience is split 50/50 on whether open source will catch up, but estimates it will happen in about six months. He reiterates that security should be the first and last pass in any LLM coding workflow.

### Personal Threats and Family Code Words [13:43](https://www.youtube.com/watch?v=yWS0udrIOc8&t=823s)
Shifting from enterprise to personal security, Yegge warns of imminent AI scams where attackers generate convincing video of a family member in distress requesting money. He urges the audience to meet with family members offline and establish code words to verify identity, noting that bank accounts will be drained.

### Q&A: Surprises in AI Coding [15:17](https://www.youtube.com/watch?v=yWS0udrIOc8&t=917s)
When asked what surprises him, Yegge says he is surprised that AI is moving faster than society. He expresses frustration that smart friends dismiss AI as a fad or "psychosis," showing tunnel vision by only looking three months backward and forward rather than seeing the exponential curve.

### Q&A: Gas Town and Beads [17:22](https://www.youtube.com/watch?v=yWS0udrIOc8&t=1042s)
Asked about Gas Town, Yegge discusses Beads, a task tracker for running Boris Cherny loops (getting agents to run unsupervised for long periods). He recounts a moment when he filed a batch of Beads tasks and they vanished—he feared a bug, but an agent had autonomously found and implemented all of them. He asks how many people regularly work with more than 10 coding agents at once, noting few do now but it will accelerate.

### Q&A: Agent Oversight and Permissions [19:26](https://www.youtube.com/watch?v=yWS0udrIOc8&t=1166s)
Yegge mentions advising friends at Tesla on the emerging space of agent oversight. He advocates for adversarial groups of supervisor agents to monitor 24/7 agents, checking whether service accounts really need all their credentials. He warns that if engineers spin up over-privileged agents, a single breach compromises everything.

### Q&A: Prompt Injection and New Security Roles [20:58](https://www.youtube.com/watch?v=yWS0udrIOc8&t=1258s)
Asked about prompt injection, Yegge confirms it is a real and growing problem on both the training and inference sides, describing it as a new form of XSRF where bad actors inject instructions like "disregard everything." He sees new "agentic security" roles emerging in companies and invites the audience member (who is already doing this work) to talk later.
