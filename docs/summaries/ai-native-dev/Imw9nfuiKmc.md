## TL;DR
Simon Maple (Tessl) and Nnenna Ndukwe (Qodo) review and improve a Qodo PR Resolver skill using Tessl's agent-based skill evaluation. The skill scores 78% on initial review against Anthropic best practices, and after automated fixes, jumps to 89%, with improvements including reduced duplication, better trigger descriptions, and added post-fix validation steps.

## Key Takeaways
- Tessl's review agent evaluates skills against Anthropic best practices, scoring them on factors like actionability, description quality, and link validity.
- The skill's description is critical metadata — it's what triggers an agent to use the skill; vague descriptions lead to skills never being invoked.
- The Qodo PR Resolver skill scored 78% initially, which is considered good (typical PR threshold is 80%+).
- Automated fixes improved the score by 11% to 89% by condensing duplicated sections, adding a shared section, and including a validate-after-applied check.
- Reducing skill length and context can sometimes improve agent adherence by making instructions easier to follow.
- AI-generated skills show uplift but benefit significantly from human validation and verification.
- Qodo uses its own skills internally — PR Resolver for fixing review issues and Get Rules for pulling coding standards into agents.
- The people and process side of adopting AI-native workflows is often harder than the technology itself, especially for experienced engineers.
- Running evals before and after skill changes can measure the real impact on task outcomes.
- Tessl review can be integrated as a GitHub Action to catch skill regressions on PRs.
