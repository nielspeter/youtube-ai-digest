# Meta made their own shadcn (Astryx)

## TL;DR
Meta has released Asterix, an MIT-licensed, fully themable React component library built on their Stylex styling solution—essentially their own version of shadcn. With over 150 accessible components refined internally over eight years, Asterix attempts to solve the design system trade-off between rigid big-company systems and copy-paste ownership by letting the system control behavior while themes control appearance at the token level.

## Key Takeaways
- Asterix is Meta's answer to shadcn, built on top of their Stylex styling solution, MIT-licensed, and reportedly refined internally for eight years across 13,000 applications.
- The library ships with over 150 accessible React components, demonstrated through templates covering tables, payment forms, kanban boards, IDE interfaces, e-commerce sites, and AI chat applications.
- Meta frames Asterix as solving a fundamental design system dilemma: adopt a rigid big-company system (like Material UI) with limited customizability, or copy-paste components (like shadcn) and own all the maintenance burden.
- Asterix separates concerns—the system controls behavior, accessibility, and quality, while themes control appearance through token-level customization.
- The `defineTheme` function generates derived tokens from minimal input (e.g., one brand color auto-generates background, accent, and secondary colors), with type-safe token definitions in JavaScript/TypeScript files.
- Customization works at three levels: global tokens, per-component overrides, and custom variants—all fully type-safe.
- Themes can be built for production, including static CSS generation for server-side rendered apps like Next.js.
- The presenter notes that a centralized theme system helps prevent the style drift that can occur with Tailwind, where developers or AI write arbitrary utility classes.
- Asterix supports multiple styling approaches including Stylex (recommended), Tailwind via a "Tailwind bridge" that maps design tokens to utility classes, Panda, and plain CSS.
- An Asterix CLI tool is designed primarily for AI agents, fetching component documentation into context and populating agents.md files, making the library highly AI-friendly even though it's new.

### Introduction and Component Overview [00:00](https://www.youtube.com/watch?v=4GaR7k9VMp4&t=0s)
The video opens by introducing Asterix as Meta's version of shadcn, built on their Stylex styling solution. ### The Design System Trade-off [01:32](https://www.youtube.com/watch?v=4GaR7k9VMp4&t=92s)
Meta's core argument for Asterix is that design systems have historically forced developers into two bad options. Option one is adopting a big-company system like Material UI, resulting in apps that look like Google products with limited customizability. Option two is using something like shadcn, where you copy and paste components into your project but then own all the code—meaning every project has its own fork and upstream fixes become hard to track. Asterix claims to end this trade-off by having the system control behavior, accessibility, and quality while themes control appearance.

### Theming with defineTheme [02:04](https://www.youtube.com/watch?v=4GaR7k9VMp4&t=124s)
The presenter demonstrates creating a custom theme using the `defineTheme` function. The color option lets you specify a single brand color (in the demo, a bright green) and automatically generates derived colors like background and accent. Typography options include a base text size (14) with a ratio (1.2), meaning each step up multiplies the base by 1.2 to generate a full type scale. Radius controls let you set everything from fully rounded to sharp corners, and a motion option controls animation speed for interactions like hovering and pressing.

### Token-Level Customization [03:38](https://www.youtube.com/watch?v=4GaR7k9VMp4&t=218s)
For finer control, the token section allows overriding specific generated tokens. Everything is type-safe because tokens are defined in JavaScript or TypeScript files rather than CSS. The presenter demonstrates changing a secondary button's background by setting the `colorNeutral` token to an array with light and dark mode values. However, changing the `colorTextPrimary` token globally affected all text on screen, which wasn't the desired effect—leading into the component-level customization feature.

### Component-Level Overrides and Custom Variants [04:09](https://www.youtube.com/watch?v=4GaR7k9VMp4&t=249s)
To scope changes to a single component, the component section lets you target specific components and variants. The presenter changes the secondary button's text color to black without affecting other components. This section also supports defining entirely custom variants—for example, a "myButton" variant with black text and red background—which then becomes a type-safe valid variant usable in the app. Pre-built themes are available as starting points, and themes can extend existing ones rather than being built from scratch each time.

### Production Builds and Brand Consistency [05:11](https://www.youtube.com/watch?v=4GaR7k9VMp4&t=311s)
Themes can be built for production and work in server-side rendered apps like Next.js, with static generation of all needed CSS and JavaScript. The presenter argues that having a single centralized theme location helps enforce brand consistency and prevent style drift—a common criticism of Tailwind where developers or AI can write arbitrary utility classes. With a set theme, changes happen in one place, are easy to diff, and AI doesn't need to reason about which Tailwind classes to use—it just uses the provided tokens and components. The presenter acknowledges you can achieve something similar with Tailwind and shadcn by defining tokens, but notes it wouldn't be type-safe and thus harder to enforce.

### Direct Component Styling and Tailwind Bridge [06:13](https://www.youtube.com/watch?v=4GaR7k9VMp4&t=373s)
Beyond theming, components can be styled directly in several ways. The recommended approach is using `xstyle` with Stylex, the system Asterix is built on. Other options include Tailwind, Panda, and plain CSS via class names. The presenter demonstrates overriding a primary button's color and background using Stylex styles passed through the `xstyle` prop. A notable feature is the "Tailwind bridge," which maps Asterix theme design tokens (colors, radius, font size) to Tailwind utility classes, letting developers continue using Tailwind while still benefiting from Asterix's theme system. The presenter personally prefers sticking with Stylex to avoid the confusion of mixing libraries.

### AI Integration and CLI [07:15](https://www.youtube.com/watch?v=4GaR7k9VMp4&t=435s)
Asterix includes a CLI tool designed primarily for AI rather than human use. It can fetch all component documentation and load it into context to help AI agents work with components, and it can populate agents.md files with guidance. The presenter found that models like Claude and GPT could work effectively with Asterisk despite it being a relatively new library. The documentation is well-written with best-practice sections on most components, which is particularly helpful for LLMs. A playground is also available for trying the library without setting up a project.
