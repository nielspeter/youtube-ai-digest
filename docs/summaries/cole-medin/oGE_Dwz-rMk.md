## TL;DR
OmniAgent is a new open-source meta-harness from Databricks that orchestrates multiple AI coding assistants—like Claude Code, Codex, and Pi—within a single, unified workflow. It enables longer, multi-agent coding tasks (e.g., implementation with Claude Code, review with Codex) with shared context, policies, guardrails, sandboxing, and cross-device collaboration, all runnable in minutes on your local machine.

## Key Takeaways
- A **meta harness** sits above individual AI coding assistants, orchestrating multiple agents to leverage each one's strengths while optimizing context and token usage.
- OmniAgent is **open source**, built by Databricks, and driven by their CTO; Databricks uses it extensively internally.
- Setup is minimal: you can hand the GitHub repo link to your AI coding assistant or run a single command; it reuses existing CLI credentials for Claude, Codex, and Pi.
- It ships with example orchestrator agents **Polly** and **Debbie**—Polly orchestrates cross-agent coding workflows (e.g., Claude implements, Codex reviews), while Debbie pits two agents against each other in a debate.
- Custom agents and orchestrators are built from three primitives: **configuration**, **skills**, and **agents**; each can have its own system prompt, executor, sandbox settings, tools, and guardrails.
- **Policies and guardrails** are defined as Python code alongside agent config, enabling human-in-the-loop approvals for dangerous actions (e.g., `git push --force`) across any orchestrated agent.
- Sandboxing options include unsandboxed, Docker, and production platforms like E2B.
- OmniAgent supports **work trees** for parallel development and can use local models via Ollama (e.g., with Pi).
- **Collaboration features** allow sharing sessions across devices (phone, desktop) over Wi-Fi or the internet, enabling team workflows.
- The harness layer matters as much as or more than the model, especially given uncertainty around access to the best LLMs (e.g., the recent "Fable 5" ban reference).
