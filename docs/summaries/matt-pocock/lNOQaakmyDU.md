# Anthropic's "dedicated monthly credit" is actually a huge cut

## TL;DR
Anthropic announced that starting June 15th, paid Claude plans will receive a "dedicated monthly credit" for programmatic (AFK) usage, separating it from human-in-the-loop usage. While framed as a bonus, this effectively slashes the usage limits for developers who rely heavily on background agents, as the new credit amounts are drastically lower than the estimated token multiplier previously available through subscriptions.

## Key Takeaways
- **The Announcement:** Starting June 15th, paid Claude plans will split usage into "human-in-the-loop" and "AFK" (away-from-keyboard) buckets, granting a specific monthly credit for programmatic AFK use cases.
- **AFK Use Cases Defined:** The credit covers the Claude Agent SDK, Claude-P (CLI), and Claude Code GitHub Actions—tools used to run Claude autonomously in the background.
- **Human-in-the-Loop Preserved:** Interactive use cases like Claude.ai, Claude Code in the terminal/IDE, and Claude Co-work are unaffected and retain their original subscription usage limits.
- **A Massive Cut in Disguise:** While technically an extra subsidy, the new dedicated credit represents a massive cut for heavy AFK users. Previously, subscriptions offered an estimated 5x to 10x token multiplier compared to standard API costs.
- **The Credit Amounts:** The monthly AFK credits mirror the subscription price: $20 for Pro, $100 for Max 5x, and $200 for Max 20x. 
- **Loss of Subsidy:** A Max 20x subscription was estimated to provide up to $5,000 worth of API credits a month. Capping it at $200 represents a roughly 96% cut for power users running background agents.
- **Increased Clarity:** Despite the drastic reduction in usage, the host appreciates the announcement because it cuts through previously confusing and contradictory terms of service regarding programmatic usage.
- **Strategic Pivot:** The host plans to keep Claude for human-in-the-loop planning but will start experimenting with OpenAI's Codex for AFK workloads, as Codex currently lacks this usage split.

The host mentions his own software factory, "Sandcastle," which orchestrates agents using Claude-P. Third is Claude Code GitHub Actions, which allows users to tag issues for Claude to work on.

### AFK vs. Human-in-the-Loop [02:39](https://www.youtube.com/watch?v=lNOQaakmyDU&t=159s)
The host draws a clear line between AFK use cases (where Claude works autonomously in the background) and human-in-the-loop use cases (where a user actively converses with the AI). Human-in-the-loop includes Claude.ai, Claude Code in the terminal, and Claude Co-work. Anthropic's new announcement firmly prioritizes human-in-the-loop usage over AFK usage.

### Why This is a Huge Cut [03:41](https://www.youtube.com/watch?v=lNOQaakmyDU&t=221s)
Before this change, a single subscription could fund both human-in-the-loop and AFK workloads. Now, Anthropic is separating them, providing a specific, smaller credit for AFK work. The host argues this is an enormous cut for developers who rely on background agents, even though it is technically a "bonus" since it provides an extra dedicated credit.

### The Math Behind the Cut [04:44](https://www.youtube.com/watch?v=lNOQaakmyDU&t=284s)
The host explains that the cut is drastic because subscriptions historically offered a massive multiplier compared to raw API costs—often estimated to be 5x to 10x cheaper. Third-party estimates suggest a 20x Max plan could provide up to $5,000 worth of API tokens a month. By limiting the AFK credit to just $200 for the 20x plan, Anthropic is drastically reducing the compute power available for background agents.

### Credit Mechanics and Clarity [05:46](https://www.youtube.com/watch?v=lNOQaakmyDU&t=346s)
The new rules state that programmatic usage draws automatically from the new credit, which resets monthly and cannot be rolled over. While this represents a huge cut, the host notes that the rules are now much clearer.
