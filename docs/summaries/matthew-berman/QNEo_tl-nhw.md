## TL;DR
The best AI users focus less on prompt engineering and more on choosing the right tokens for the right tasks. By understanding that models vary in intelligence density, speed, and cost, you can use a multi-model workflow—using expensive frontier models for planning and reviewing, and cheap, fast models for execution—to get higher quality results at a fraction of the price.

## Key Takeaways
- Not all tokens are equal: they vary in quality, speed, and cost, and the goal is to use the right mixture for the specific task.
- A model's raw price per million tokens can be deceiving; what matters is the actual cost to complete a task, which depends on "intelligence density."
- A cheaper model like Kimmy K3 might cost half the price of GPT 5.6 Soul per token, but if it takes twice as many tokens to solve the same problem, the overall cost is the same.
- Different AI models approach problems differently and excel at different things; letting two different models review each other's code finds more bugs than self-review.
- Output speed matters: if a cheaper model takes twice as long to complete a task, you are losing valuable time.
- A highly effective workflow involves using a frontier model (like Fable) for planning, a cheap/fast model (like Grock 4.5 or Cursor's composer) for code execution, and a different frontier model (like GPT 5.6) for final review.
- The execution phase is the most expensive because it generates the most output tokens, which cost significantly more than input tokens.
- Using a mixed-model workflow can reduce the cost of a complex task from $81 (using a single expensive model) to $25.55 while maintaining high quality.
- The future of AI pricing is a battle between closed-source and open-source models; if open-source wins, profit margins on tokens will drop, benefiting consumers and application developers.
- As the price of tokens decreases, usage increases, shifting profits in the AI stack toward chip providers (like Nvidia), hyperscalers, and the application layer.

## Detailed Breakdown

### Tokens and Intelligence Density [00:00](https://www.youtube.com/watch?v=QNEo_tl-nhw&t=0s)
The video opens by explaining that expert AI users care less about prompt engineering and more about selecting the right tokens. A token is the basic unit of a large language model, roughly analogous to a word, though a single word can be made of multiple tokens. Models predict the next token, but not all tokens are equal in quality, speed, or price. The key concept is "intelligence density"—how much intelligence is packed into each individual token.

### The True Cost of Completing Tasks [02:06](https://www.youtube.com/watch?v=QNEo_tl-nhw&t=126s)
New models are often judged by their pricing per million tokens. For example, GPT 5.6 Soul costs $5 per million input tokens and $30 per million output tokens. A new open-source Chinese model, Kimmy K3, is half that price. However, raw token price is misleading. On benchmarks, Kimmy K3 and GPT 5.6 Soul cost roughly the same to complete an average task ($0.95 vs $1.04), while Claude Fable 5 is much more expensive ($2.75). This is because Kimmy K3 requires twice as many tokens to think through and solve a problem as GPT 5.6, effectively canceling out the per-token savings.

### Cross-Model Code Review [05:42](https://www.youtube.com/watch?v=QNEo_tl-nhw&t=342s)
Different models behave differently and excel at different tasks. The video's sponsor, Greile, released a feature demonstrating that letting two different models review each other's code is highly effective. For instance, when Claude authors code and GPT 5.5 reviews it, GPT 5.5 finds more bugs than Claude would reviewing its own code. Different models are prone to different types of errors, making cross-model review a powerful technique for shipping high-quality code.

### The Multi-Model Workflow [07:50](https://www.youtube.com/watch?v=QNEo_tl-nhw&t=470s)
To save money and maximize quality, the host recommends a three-step workflow using a mixture of models. First, use the best, most expensive frontier model (e.g., Fable) for the planning phase, as high-level thinking is the hardest part. Second, use a cheap and fast model (e.g., Grock 4.5 or Cursor's composer) for the execution phase (writing the code), since following a plan is technically easier. Third, use a different frontier model (e.g., GPT 5.6) to review the completed code against the original spec. This approach is faster and dramatically cheaper.

### Token Economics in the Workflow [09:22](https://www.youtube.com/watch?v=QNEo_tl-nhw&t=562s)
The workflow is cost-effective because of how input and output tokens are priced. The planning stage uses many input tokens (reading the codebase) but few output tokens (writing a plan). The execution stage uses fewer input tokens (reading the plan) but many expensive output tokens (writing all the code). The review stage flips back to mostly input tokens. Because the expensive execution phase is handled by a cheap model, the overall cost drops significantly. A task that costs $81 using Fable alone or $46.50 using GPT 5.6 alone costs just $25.55 using the mixed-model approach.

### The Future of AI Pricing: Open vs. Closed Source [12:04](https://www.youtube.com/watch?v=QNEo_tl-nhw&t=724s)
The future of AI intelligence pricing is being decided now through a battle between closed-source and open-source models. Closed-source labs (OpenAI, Anthropic) sell tokens with high margins and have little incentive to drop prices. Open-source models like Kimmy K3 can be downloaded and optimized by anyone, meaning hyperscalers will compete to serve them at the lowest possible price. If open-source wins, profit margins on tokens will vanish, shifting profits to chip providers (Nvidia), hyperscalers, and the application layer, while benefiting consumers through lower costs and increased token usage.
